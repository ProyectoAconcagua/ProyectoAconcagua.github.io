Punto de Venta

l10n_ar_point_of_sale
Reportar Bug
  • Descripción
  • Capturas

El módulo de punto de venta introduce conceptos básicos de la contabilidad Argentina, tales como denominación, punto de venta, tipo de factura, tipo de comprobante, etc..

Contiene la lógica necesaria para:

  • Asignar la denominación correspondiente según las posiciones fiscales
  • Calcular el número de comprobante según tipo, denominación y punto de venta
  • Realizar chequeos fiscales previos a validar la factura, como por ejemplo, que coincidan la posición fiscal y la denominación del comprobante.
  • Incluir el concepto de notas de débito sin afectar demasiado el código de OpenERP/Odoo
  • Posibilidad de agregar impuestos en forma manual a los comprobantes Facturas, NC y ND
  • Chequeo de duplicado de factura
  • Chequeo de formato del número de factura
  • Solo realiza chequeos fiscales a comprobantes locales. Permite comprobantes con Clientes/Proveedores del exterior.

Cobros y pagos

l10n_ar_account_payment
Reportar Bug
  • Descripción
  • Capturas

El módulo de cobros y pagos extiende el módulo account_voucher para agregarle varias formas de cobros/pagos. Soporta multimoneda.

Además, agrega varios hooks para extender la funcionalidad en los comprobantes de cobros y pagos, por lo tanto, sirve de base para los módulos de cheques y retenciones.

Contiene la lógica necesaria para:

  • Configurar distintas formas de cobro y de pago por separado.
  • Asignar una cuenta contable a cada forma de cobro/pago.
  • Incluir de forma automática las formas de cobro/pago según sea para cobro o pago y tipo de moneda.
  • Crear asientos contables correspondientes según la configuración efectuada.

Este módulo carga en el sistema las plantillas correspondientes a cuentas contables, impuestos y códigos de impuestos para un Plan de Cuentas de uso básico en Argentina.

Chequeo de CUIT/CUIL

base_vat_ar
Reportar Bug
  • Descripción
  • Capturas

Este módulo agrega la lógica de chequeo de CUIT/CUIL para Argentina, extendiéndo el módulo base_vat que viene con OpenERP/Odoo.

Cambia la lógica que utiliza de base el sistema para elegir la función de chequeo. Al instalar este módulo ya no es necesario agregar el código de país delante del número de CUIT/CUIL, sino que se utiliza el país del cliente/proveedor para saber a qué país pertenece.

Características destacables:

  • Algoritmo de chequeo de CUIT/CUIL.
  • Cambio de lógica para detectar el país para el cual chequear, no hay necesidad de agregar AR o cualquier otro código de país delante del número.
  • Se agregan tipos de documentos y se precargan. Como ser CUIT, CUIL, DNI, CI, LE, etc.
  • Sobre cada tipo de documento se puede configurar si se debe realizar o no el chequeo.

Gestión de Cheques

l10n_ar_account_check
Reportar Bug
  • Descripción
  • Capturas

Este módulo gestiona cheques propios y de terceros. Hereda del módulo de Cobros y Pagos para agregar a los recibos la posibilidad de cargar cheques de terceros recibidos y a las ordenes de pago asignar cheques propios o de terceros ya cargados en el sistema.

Posee varias vistas de calendario por diferentes fechas, de modo que se pueda tener un panorama de cuando depositar un cheque, o cuando un Proveedor puede llegar a depositar un cheque propio.

Características principales:

  • Adaptación al módulo de cobros y pagos de forma que los cheques que se incluyan en el comprobante, suman al total del cobro/pago.
  • Manejo de estados de cheques de tercero (En cartera, depositado, entregado, rechazado).
  • Vistas de calendario con información temporal de los cheques.
  • Generación de las líneas del asiento contable en el Recibo/Orden de Pago correspondiente a la configuración.
  • Wizard de depósito que genera el asiento contable correspondiente.
  • Wizard de rechazo de cheque con generación automática de nota de débito por el monto del cheque más gastos bancarios.

Gestión de chequeras

l10n_ar_account_create_check
Reportar Bug
  • Descripción
  • Capturas

Este módulo agrega una gestión de chequeras para los cheques propios que se adapta al módulo de cobros y pagos y el de gestión de cheques para su uso normal.

Permite una carga más rápida de los cheques propios en ordenes de pago, ya que, solo debe ingresarse el número de cheque y si está disponible en alguna de las chequeras, se cargan los datos ya configurados en la misma.

Características destacables:

  • Wizard de creación de chequeras asociadas a una cuenta bancaria.
  • Carga rápida en órdenes de pago.
  • Gestión de las chequeras, sabiendo qué cheques están disponibles en cada chequera y cuál fue el destino de los cheques ya emitidos.

Retenciones

l10n_ar_retentions_basic
Reportar Bug
  • Descripción
  • Capturas

Agrega el concepto de Retenciones tal como se utilizan en Argentina, adaptándolo a la lógica de impuestos que OpenERP/Odoo tiene de base.

Permite cargar en forma manual las retenciones aplicadas en los pagos y las retenciones sufridas en los cobros, realizando las líneas contables correspondientes, según la configuración de las retenciones.

Características destacables:

  • Adaptación al sistema sin romper la lógica que trae de base.
  • Diferenciación entre retenciones a aplicadas y sufridas.
  • Extensión del módulo de cobros y pagos para agregar la carga de retenciones.
  • Menúes con las retenciones sufridas y aplicadas que facilita la generación de informes de retenciones.
  • Precarga de retenciones comúnmente utilizadas, con la configuración de impuestos correspondiente que ayuda al usuario a cargar nuevas retenciones utilizando estas como modelo.
